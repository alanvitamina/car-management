## 2. 页面定稿原则

- 飞书内嵌访问优先，兼顾桌面浏览器
- 第一版优先闭环效率，不做复杂装饰性交互
- 所有主动作必须满足“状态可见、结果可见、失败有提示”
- 所有关键业务按钮必须有明确的角色和状态前置条件

## 4. 导航与入口定稿

### 4.1 工作台导航

工作台根据角色动态展示快捷入口：

- 员工：发起申请、我的申请、我的补助
- 审批人：待我审批、我的申请
- 行政经理：待派车、私车核验、车辆管理、司机管理、核算管理、数据看板
- 司机：我的出车任务、我的补助
- 系统管理员：系统配置、集成日志、规则配置

### 4.2 详情页统一入口

所有飞书通知、工作台待办和列表操作，最终都应能跳到统一的申请详情页或任务详情页，避免入口分裂。

## 5. 页面级定稿说明

### 5.1 `P-01` 工作台

页面目标：

- 提供统一落地页
- 根据角色展示待办和快捷入口

主要模块：

- 用户欢迎区
- 快捷入口区
- 待办摘要区
- 申请状态摘要区
- 车辆状态摘要区
- 最近申请或最近任务列表

关键字段：

- 当前用户姓名
- 角色标签
- 待审批数量
- 待派车数量
- 待确认数量
- 待核验数量
- 空闲车辆数
- 已预占车辆数
- 出车中车辆数

交互规则：

- 角色无权限时隐藏快捷入口，不展示禁用按钮
- 数据接口失败时展示“刷新重试”
- 无待办时展示空状态卡片，不展示空白表格

### 5.2 `P-02` 发起申请

页面目标：

- 员工完成公务用车或私车公用申请

表单字段：

| 字段 | 组件建议 | 必填 | 说明 |
| --- | --- | --- | --- |
| 申请类型 | 单选 | 是 | 公务用车/私车公用 |
| 申请人 | 只读文本 | 是 | 自动带出 |
| 所属部门 | 只读文本 | 是 | 自动带出 |
| 开始时间 | 日期时间选择器 | 是 | 需早于结束时间 |
| 结束时间 | 日期时间选择器 | 是 | 需晚于开始时间 |
| 用车事由 | 文本框 | 是 | 最长建议 200 字 |
| 目的地 | 文本框 | 是 | 最长建议 100 字 |
| 备注 | 多行文本 | 否 | 最长建议 500 字 |

附加展示：

- 一级审批人
- 二级审批人
- 公务用车可用车辆摘要

校验规则：

- 开始时间、结束时间必填
- 开始时间必须早于结束时间
- 用车事由、目的地不得为空
- 提交前必须成功带出审批人

按钮规则：

- `保存草稿`：始终可见
- `提交申请`：校验通过后可点击

动态交互：

- 切换申请类型时重置与另一类型无关的提示区
- 公务用车时显示“可用车辆摘要”
- 私车公用时隐藏“可用车辆摘要”
- 时间变化后自动刷新可用车辆摘要

异常处理：

- 审批人带出失败：阻止提交，提示联系管理员
- 飞书审批创建失败：提交后提示“申请已创建但审批发起异常”或按整体失败策略处理

### 5.3 `P-03` 我的申请列表

筛选条件：

- 申请类型
- 当前状态
- 时间范围

列表字段：

- 申请单号
- 申请类型
- 开始时间
- 结束时间
- 目的地
- 当前状态
- 最近更新时间
- 操作

操作规则：

- `详情`：始终可见
- `取消`：仅允许取消状态可见
- `发起变更`：仅非终态可见
- `重新发起`：驳回、取消、完成状态可见

空状态：

- 无数据时展示“暂无申请记录”

### 5.4 `P-04` 申请详情

页面目标：

- 统一展示一张申请单的全生命周期

区块结构：

- 基本信息
- 审批进度
- 派车信息
- 消耗/里程信息
- 核算结果
- 操作区
- 操作日志

按钮可见规则：

| 按钮 | 可见角色 | 可见状态 |
| --- | --- | --- |
| 取消申请 | 申请人/行政经理/管理员 | `DRAFT` `PENDING_L1` `PENDING_L2` `PENDING_DISPATCH` `RESERVED` |
| 发起变更 | 申请人/行政经理 | 非终态且未执行完成 |
| 重新发起 | 申请人/行政经理 | `REJECTED` `CANCELLED` `COMPLETED` |
| 去派车 | 行政经理 | `PENDING_DISPATCH` |
| 去确认消耗 | 用车人/行政经理 | 公务用车且 `PENDING_CONFIRM` |
| 去私车核验 | 行政经理 | 私车公用且 `PENDING_CONFIRM` |

展示规则：

- 审批进度使用时间轴或步骤条
- 审批记录、驳回意见、移交记录必须可见
- 公务用车未派车前，派车区显示“待派车”
- 私车公用不展示车辆司机信息区块或展示为“无派车环节”

### 5.5 `P-05` 待派车列表

筛选条件：

- 申请时间范围
- 申请人
- 部门

列表字段：

- 单号
- 申请人
- 部门
- 用车时间
- 目的地
- 可用车辆数
- 可用司机数
- 当前状态
- 操作

交互规则：

- 仅展示 `PENDING_DISPATCH` 状态数据
- 当可用车辆数或可用司机数为 0 时高亮提醒
- `去派车` 按钮点击进入派车详情页

### 5.6 `P-06` 派车详情

页面目标：

- 完成车辆、司机分配与资源预占

页面区块：

- 申请信息
- 候选车辆列表
- 候选司机列表
- 冲突校验结果
- 派车备注

核心字段：

- 车辆 ID / 车牌号 / 车型 / 状态 / 冲突原因
- 司机 ID / 姓名 / 当前状态 / 冲突原因

交互规则：

- 选择车辆后可立即触发冲突校验
- 选择司机后可立即触发冲突校验
- 点击 `确认派车` 前必须再次调用后端二次校验
- 派车成功后返回申请详情页并提示成功

按钮规则：

- `确认派车`：当已选车辆且已选司机且后端校验通过时可点击
- `改派`：仅已有派车记录且尚未执行时可见

异常处理：

- 若提交时资源已被占用，保持在当前页并展示明确冲突原因

### 5.7 `P-07` 我的出车任务

列表字段：

- 单号
- 申请人
- 用车时间
- 目的地
- 当前状态
- 操作

交互规则：

- 默认展示司机本人任务
- 仅对待录入或被驳回的任务显示“录入消耗”

### 5.8 `P-08` 公务用车消耗录入

表单字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 出车前里程照片 | 是 | 里程表照片，后端自动提取 EXIF 拍摄时间作为实际出车时间 |
| 出车前里程 | 是 | 数值型，司机看照片手动输入 |
| 收车后里程照片 | 是 | 里程表照片，后端自动提取 EXIF 拍摄时间作为实际收车时间 |
| 结束后里程 | 是 | 数值型，司机看照片手动输入 |
| 实际出车时间 | 否 | 只读，从出车前里程照片 EXIF 自动提取；EXIF 缺失时显示时间输入框 |
| 实际收车时间 | 否 | 只读，从收车后里程照片 EXIF 自动提取；EXIF 缺失时显示时间输入框 |
| 自动计算里程 | 是 | 只读，= 结束后里程 - 出车前里程 |
| 油费 | 否 | 数值型，默认 0 |
| 路桥费 | 否 | 数值型，默认 0 |
| 停车费 | 否 | 数值型，默认 0 |
| 其他费用 | 否 | 数值型，默认 0 |
| 备注 | 否 | 补充说明 |

交互规则：

- 上传出车前里程照片后，后端返回提取的 EXIF 拍摄时间，前端回填"实际出车时间"字段
- 上传收车后里程照片后，后端返回提取的 EXIF 拍摄时间，前端回填"实际收车时间"字段
- EXIF 缺失时，实际出车/收车时间字段切换为可编辑，司机手工选择时间
- 照片上传后司机仍可手动修改时间，系统记录 `time_source = MANUAL`

校验规则：

- 出车前里程照片、收车后里程照片必须上传
- 结束后里程不得小于出车前里程
- 费用不得为负数
- 实际收车时间不得早于实际出车时间

提交后结果：

- 状态进入 `PENDING_CONFIRM`
- 实际出车/收车时间同步写入 `car_dispatch_record`
- 通知用车人确认

### 5.9 `P-09` 公务用车消耗确认

页面目标：

- 用车人确认或驳回司机提交的消耗记录

展示内容：

- 任务基本信息
- 司机录入里程
- 费用明细
- 附件列表
- 历史驳回记录

按钮规则：

- `确认`：申请人为当前用户时可见
- `驳回`：申请人为当前用户或行政经理时可见

交互规则：

- 驳回必须填写原因
- 确认后数据锁定
- 确认后跳回详情页并展示核算结果入口

### 5.10 `P-10` 私车公用录入

表单字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 出车前里程照片 | 是 | 里程表照片，后端自动提取 EXIF 拍摄时间记录实际出发时间 |
| 出车前里程 | 是 | 数值，员工看照片手动输入 |
| 收车后里程照片 | 是 | 里程表照片，后端自动提取 EXIF 拍摄时间记录实际到达时间 |
| 结束后里程 | 是 | 数值，员工看照片手动输入 |
| 起点 | 是 | 文本 |
| 终点 | 是 | 文本 |
| 自动计算总里程 | 是 | 只读，= 结束后里程 - 出车前里程 |
| 路桥费 | 否 | 数值 |
| 停车费 | 否 | 数值 |
| 其他费用 | 否 | 数值 |
| 备注 | 否 | 说明 |

交互规则：

- 上传出车前里程照片后，后端返回提取的 EXIF 拍摄时间，前端回填”实际出发时间”
- 上传收车后里程照片后，后端返回提取的 EXIF 拍摄时间，前端回填”实际到达时间”
- EXIF 缺失时页面展示时间输入框
- 里程照片在核验页面对行政经理可见，作为里程校核的辅助依据

校验规则：

- 出车前里程照片、收车后里程照片必须上传
- 起点、终点必须填写
- 结束后里程不得小于出车前里程

提交结果：

- 主状态维持或进入 `PENDING_CONFIRM`
- 生成”待核验”任务并通知行政经理核验

### 5.11 `P-11` 私车公用核验

页面目标：

展示字段：

- 申请人
- 时间
- 起点
- 终点
- 出车前里程照片（可查看，辅助校核）
- 收车后里程照片（可查看，辅助校核）
- 记录总里程
- 导航核验里程
- 偏差值
- 核算里程
- 路桥费
- 停车费
- 预计补助
- 核验说明

交互规则：

- 导航核验里程默认为空，由行政经理录入
- 当偏差超过默认阈值时高亮显示
- 核算里程支持三种来源：记录总里程、导航核验里程、手工调整
- 手工调整时必须填写核验说明
- 可点击查看申请人上传的里程照片原图，用于对比里程表读数与录入数据

按钮规则：

- `退回修改`：可见，需填写原因
- `确认核验`：可见，需填写导航核验里程和核算里程

结果：

- 确认后自动生成私车公用补助结果
- 主状态进入 `COMPLETED`

## 6. 状态展示规范

### 6.1 申请状态文案

建议前端统一展示文案如下：

| 状态值 | 展示文案 |
| --- | --- |
| `DRAFT` | 草稿 |
| `PENDING_L1` | 待一级审批 |
| `PENDING_L2` | 待二级审批 |
| `REJECTED` | 审批驳回 |
| `PENDING_DISPATCH` | 待派车 |
| `RESERVED` | 已派车待执行 |
| `IN_PROGRESS` | 执行中 |
| `PENDING_CONFIRM` | 待确认/待核验 |
| `COMPLETED` | 已完成 |
| `CANCELLED` | 已取消 |
| `APPROVAL_EXCEPTION` | 审批异常 |

说明：

- 对私车公用而言，`PENDING_CONFIRM` 在页面上可按当前任务进一步展示为“待录入”或“待核验”，但后端主状态值保持不变

### 6.2 颜色建议

- 待处理类：蓝色
- 成功完成类：绿色
- 驳回/异常类：红色
- 取消类：灰色
- 执行中类：橙色

## 7. 通用交互规范

### 7.1 表单提交规范

- 提交中按钮需禁用，避免重复提交
- 提交成功后需有全局成功反馈
- 提交失败需展示明确原因，不允许只弹“系统错误”

### 7.2 列表规范

- 列表支持分页
- 列表筛选条件应可重置
- 无数据时必须有空状态提示

### 7.3 异常提示规范

- 资源冲突：展示具体冲突对象和原因
- 审批异常：展示“审批发起异常，请联系管理员或稍后重试”
- 权限不足：展示“当前角色无权操作”

## 8. 移动端与飞书内嵌要求

- 表单页面在移动宽度下采用单列布局
- 重要按钮固定在底部操作栏或页面底部
- 列表页在窄屏场景可退化为卡片列表
- 不依赖鼠标悬停展示关键信息
