# 公务用车管理系统：业务规则清单 V1

## 1. 文档说明

本文档用于统一收口第一期公务用车管理系统的业务规则，作为产品、开发、测试、实施共同遵循的“规则字典”。

在文档冲突时，本文件用于回答以下问题：

- 某个动作能不能做
- 某个状态能不能流转
- 某类数据该由谁录、谁看、谁确认
- 某项补助到底按什么规则算

## 2. 使用原则

### 2.1 规则优先级

规则优先级从高到低如下：

1. 法规、公司制度或审计要求
2. 本文档的冻结规则
3. 页面交互文档
4. 接口实现细节

### 2.2 规则编码建议

为便于评审、测试和缺陷追踪，建议所有规则使用稳定编码：

- `APP`：申请规则
- `APR`：审批规则
- `DSP`：派车规则
- `EXE`：执行与消耗规则
- `SUB`：补助核算规则
- `PERM`：权限规则
- `MSG`：通知规则
- `AUD`：审计与追溯规则

## 3. 核心字典定义

### 3.1 申请类型

| 编码 | 名称 | 说明 |
| --- | --- | --- |
| `OFFICIAL` | 公务用车 | 使用公司公务车辆，由行政经理派车 |
| `PRIVATE` | 私车公用 | 使用员工私家车，事后核验里程并核算补助 |

### 3.2 申请单状态

| 状态值 | 名称 | 说明 |
| --- | --- | --- |
| `DRAFT` | 草稿 | 未提交审批 |
| `PENDING_L1` | 待一级审批 | 已提交，待一级审批人处理 |
| `PENDING_L2` | 待二级审批 | 一级通过，待二级审批人处理 |
| `REJECTED` | 审批驳回 | 任一审批节点驳回 |
| `PENDING_DISPATCH` | 审批通过待派车 | 公务用车审批通过后待分配资源 |
| `RESERVED` | 已预占待执行 | 已派车且资源已锁定 |
| `IN_PROGRESS` | 执行中 | 已实际出车或执行中 |
| `PENDING_CONFIRM` | 待确认/待核验 | 公务用车待用车人确认，私车公用待行政核验 |
| `COMPLETED` | 已完成 | 消耗确认或核算完成 |
| `CANCELLED` | 已取消 | 申请终止且不再继续 |
| `APPROVAL_EXCEPTION` | 审批异常 | 飞书审批实例创建或回调异常待补偿 |

### 3.3 审批状态

| 状态值 | 名称 | 说明 |
| --- | --- | --- |
| `PENDING` | 待审批 | 节点待处理 |
| `APPROVED` | 已通过 | 节点审批通过 |
| `REJECTED` | 已驳回 | 节点审批驳回 |
| `WITHDRAWN` | 已撤回 | 申请人主动撤回审批 |
| `TRANSFERRED` | 已移交 | 节点被移交给他人处理 |
| `ADD_SIGNING` | 加签中 | 存在加签待完成 |
| `COUNTER_SIGNING` | 会签中 | 存在会签待完成 |

### 3.4 车辆状态

| 状态值 | 名称 | 说明 |
| --- | --- | --- |
| `IDLE` | 空闲 | 可被分配 |
| `RESERVED` | 已预占 | 已派车待出车 |
| `IN_USE` | 出车中 | 正在执行任务 |
| `MAINTENANCE` | 维修中 | 不可派 |
| `DISABLED` | 停用 | 不可派 |

### 3.5 司机状态

| 状态值 | 名称 | 说明 |
| --- | --- | --- |
| `AVAILABLE` | 在岗可派 | 可被分配 |
| `RESERVED` | 已预占 | 已派车待出车 |
| `IN_USE` | 出车中 | 正在执行任务 |
| `REST` | 休息 | 不可派 |
| `LEAVE` | 请假 | 不可派 |
| `DISABLED` | 停用 | 不可派 |

## 4. 权限与数据范围规则

### 4.1 `PERM-001` 角色边界规则

- 普通员工可发起申请、查看本人申请、取消本人允许取消的申请、确认本人用车消耗、录入本人私车公用数据
- 一级审批人只处理分配给自己的一级审批任务
- 二级审批人只处理分配给自己的二级审批任务
- 行政经理负责派车、改派、资源释放、私车公用核验、档案维护、规则维护
- 行政司机只查看分配给自己的任务并提交公务用车消耗记录
- 系统管理员负责系统配置、组织同步、集成日志、异常排查，不默认参与业务审批

### 4.2 `PERM-002` 数据可见范围规则

- 普通员工默认只看与本人相关的数据
- 司机默认只看本人任务与本人补助相关数据
- 一级审批人可看本人待审批和已审批记录，不默认看全公司申请
- 行政经理可看全部业务申请、派车、核验、核算数据
- 系统管理员可看全局配置和日志，也可因运维需要查看业务单，但不应拥有“代审批”默认能力

### 4.3 `PERM-003` 敏感数据控制规则

- 补助金额、费用明细、核算依据只对申请人本人、行政经理、系统管理员开放
- 审批人在审批阶段默认看申请必要字段，不默认展示执行后费用明细

## 5. 申请规则

### 5.1 `APP-001` 申请类型规则

- 每张申请单只能对应一种申请类型
- 公务用车和私车公用不能混在同一张申请单中

### 5.2 `APP-002` 必填字段规则

提交申请时必须填写：

- 申请类型
- 开始时间
- 结束时间
- 用车事由
- 目的地

备注为选填。

### 5.3 `APP-003` 时间合法性规则

- `start_time` 必须早于 `end_time`
- 不允许开始时间等于结束时间
- 时间格式统一为 `yyyy-MM-dd HH:mm:ss`

### 5.4 `APP-004` 草稿规则

- 草稿状态允许保存和编辑
- 草稿状态不触发审批、不占用资源、不发通知

### 5.5 `APP-005` 审批人带出规则

- 一级审批人默认取申请人所属部门负责人
- 二级审批人默认取行政经理
- 审批人由后端计算并回写，前端只做展示
- 组织数据缺失时，申请不能直接提交，需进入异常提示或人工处理

### 5.6 `APP-006` 申请提交规则

- 申请提交成功后，系统必须立即创建业务单
- 业务单创建成功后应自动发起飞书审批实例
- 飞书审批创建失败时，申请单进入 `APPROVAL_EXCEPTION`，不得假装已进入正常审批

### 5.7 `APP-007` 取消规则

- `DRAFT`、`PENDING_L1`、`PENDING_L2`、`PENDING_DISPATCH`、`RESERVED` 状态允许取消
- `IN_PROGRESS` 不允许普通员工取消
- `COMPLETED`、`CANCELLED` 不允许再次取消
- 取消必须记录取消原因

### 5.8 `APP-008` 变更规则

- 变更不覆盖原申请单
- 变更应生成新申请单，并与原申请建立关联
- 涉及时间、地点、类型、资源占用的变更，必须重新审批

### 5.9 `APP-009` 重新发起规则

- 支持基于原申请复制生成新申请
- 新申请必须生成新单号、新审批实例
- 原单和新单需保留关联链路，便于追溯

## 6. 审批规则

### 6.1 `APR-001` 审批承载边界规则

- 审批动作发生在飞书
- 审批主状态由新系统维护
- 飞书审批只承载审批协同，不承载业务执行数据

### 6.2 `APR-002` 两级审批规则

- 第一阶段默认两级审批
- 一级通过后才能进入二级审批
- 任一级驳回均结束当前申请主流程

### 6.3 `APR-003` 审批驳回规则

- 一级驳回：`PENDING_L1 -> REJECTED`
- 二级驳回：`PENDING_L2 -> REJECTED`
- 驳回后申请单不可继续派车

### 6.4 `APR-004` 审批通过规则

- 公务用车二级审批通过后：`PENDING_L2 -> PENDING_DISPATCH`
- 私车公用二级审批通过后：进入执行/录入阶段，主状态统一落为 `PENDING_CONFIRM`

说明：

- 私车公用没有派车环节，因此不进入 `PENDING_DISPATCH`
- 对私车公用而言，`PENDING_CONFIRM` 表示“待录入/待核验的统一业务阶段”，具体当前动作由页面任务和操作记录区分

### 6.5 `APR-005` 审批撤回规则

- 申请人可在审批过程中发起撤回
- 撤回后主申请单进入 `CANCELLED`
- 撤回动作必须保留在审批记录中

### 6.6 `APR-006` 移交、加签、会签规则

- 移交、加签、会签记录在审批记录表中
- 这些动作不直接改变主申请单状态名称
- 节点完成条件由审批记录判断，主申请状态只在“节点完成”后推进

### 6.7 `APR-007` 审批回调幂等规则

- 同一审批回调事件重复推送时不得重复推进主状态
- 原始回调报文必须保留
- 幂等判断建议基于外部事件唯一键或审批实例 ID + 动作时间 + 审批人组合键

## 7. 派车与资源规则

### 7.1 `DSP-001` 派车前置规则

- 仅 `PENDING_DISPATCH` 状态允许派车
- 派车前必须二次校验车辆和司机是否可用

### 7.2 `DSP-002` 车辆可用性规则

车辆可用必须同时满足：

- 状态为 `IDLE`
- 在申请时间段内不存在其他有效预占或出车任务
- 不处于维修中或停用状态

### 7.3 `DSP-003` 司机可用性规则

司机可用必须同时满足：

- 状态为 `AVAILABLE`
- 在申请时间段内不存在其他有效预占或出车任务
- 不处于休息、请假或停用状态

### 7.4 `DSP-004` 预占规则

- 派车成功后，申请状态进入 `RESERVED`
- 同步把车辆和司机状态更新为 `RESERVED`
- 派车结果应通知申请人和司机

### 7.5 `DSP-005` 改派规则

- 在未实际执行前，行政经理可以改派车辆和司机
- 改派前必须重新做冲突校验
- 改派后必须保留原派车记录与新派车记录
- 执行中不支持直接改派

### 7.6 `DSP-006` 释放规则

以下场景必须释放资源：

- 已预占申请被取消
- 派车后改派前先释放旧资源
- 异常终止后经人工确认无需继续执行

## 8. 执行与消耗规则

### 8.1 `EXE-001` 公务用车执行主体规则

- 公务用车由司机提交消耗记录
- 用车人负责确认或驳回消耗记录

### 8.2 `EXE-002` 私车公用执行主体规则

- 私车公用由申请人本人录入起止地点、起始里程、结束里程和费用
- 行政经理负责导航核验与核算里程确认

### 8.3 `EXE-003` 里程计算规则

- 公务用车和私车公用的记录总里程统一按 `结束后里程 - 出车前里程` 自动计算
- 若计算结果小于 0，则禁止提交

### 8.4 `EXE-004` 费用填写规则

- 油费、路桥费、停车费、其他费用允许为 0
- 费用项不得为负数

### 8.5 `EXE-005` 附件规则

- 第一阶段支持上传电子附件
- 对存在报销费用的记录，页面应支持附件上传
- 第一阶段不强制做票据真伪校验

### 8.6 `EXE-006` 公务用车确认规则

- 用车人确认后，消耗记录锁定
- 确认后自动触发司机补助核算
- 确认后自动恢复车辆和司机为可用状态

### 8.7 `EXE-007` 公务用车驳回规则

- 用车人发现问题可驳回消耗记录
- 驳回必须填写原因
- 驳回后司机可修改并重新提交

### 8.8 `EXE-008` 私车公用核验规则

- 行政经理必须录入导航核验里程或等价核验结果
- 最终补助核算只能使用“核算里程”，不能直接用记录总里程替代
- 若导航核验里程与记录总里程偏差较大，页面需高亮提醒

## 9. 补助核算规则

### 9.1 `SUB-001` 私车公用基础规则

- 私车公用补助按单次确认后的核算里程计算
- 过路过桥费、停车费据实报销
- 不再单独核算油耗和车耗补贴

### 9.2 `SUB-002` 私车公用分档规则

- 单次核算总里程小于等于 `100` 公里时，按 `0.8 元/公里`
- 单次核算总里程大于 `100` 公里时，按 `1 元/公里`

### 9.3 `SUB-003` 司机补助季节规则

- 冬季时间范围为每年 `10 月 1 日` 至次年 `4 月 30 日`
- 夏季时间范围为每年 `5 月 1 日` 至 `9 月 30 日`

### 9.4 `SUB-004` 司机补助工作日时段规则

- 冬季工作日加班区间：早间小于 `08:30`、午间 `12:00-13:00`、晚间大于 `17:30`
- 夏季工作日加班区间：早间小于 `08:30`、午间 `12:00-13:30`、晚间大于 `18:00`
- 工作日补助另加早出 `1` 小时和晚归 `1` 小时

### 9.5 `SUB-005` 周末及节假日规则

- 周末及节假日全部用车时长计入加班
- 周末和节假日均另加出车前 `1` 小时和收车后 `1` 小时
- 工作日按 `1` 倍计算
- 周末按 `2` 倍计算
- 法定节假日按 `3` 倍计算

### 9.6 `SUB-006` 规则快照规则

- 所有核算结果都必须保存规则版本和计算依据
- 补助规则后续修改不得影响历史已确认单据

### 9.7 `SUB-007` 金额精度规则

- 第一阶段金额默认保留两位小数
- 若财务最终要求特殊取整方式，作为后续变更处理

## 10. 通知规则

### 10.1 `MSG-001` 审批通知规则

- 申请提交后通知一级审批人
- 一级审批通过后通知二级审批人
- 审批结果通知申请人

### 10.2 `MSG-002` 派车通知规则

- 派车成功后通知申请人和司机
- 改派成功后再次通知申请人和新司机

### 10.3 `MSG-003` 执行通知规则

- 公务用车消耗提交后通知用车人确认
- 消耗被驳回后通知司机补录
- 私车公用提交后通知行政经理核验

### 10.4 `MSG-004` 催办规则

- 长时间未审批、未确认、未核验的任务允许由定时任务催办
- 第一阶段催办以飞书通知为主

## 11. 审计与追溯规则

### 11.1 `AUD-001` 操作留痕规则

以下动作必须留痕：

- 提交申请
- 取消申请
- 发起变更
- 审批通过/驳回/撤回/移交/加签/会签
- 派车
- 改派
- 释放资源
- 提交消耗
- 驳回消耗
- 确认消耗
- 核验私车公用
- 生成补助结果

### 11.2 `AUD-002` 外部报文留痕规则

- 飞书审批创建请求与响应建议记录摘要
- 飞书审批回调原始报文必须保留
- 回调解析失败不得直接丢弃原始数据

### 11.3 `AUD-003` 历史不可覆盖规则

- 申请单主状态只能流转，不能无痕覆盖
- 审批记录、派车记录、核算记录必须保留历史版本或历史明细

## 12. 开发与测试使用建议

开发和测试在使用本文档时建议遵循：

1. 每个接口、页面动作都应能映射到一条或多条规则编码
2. 每个测试用例至少引用一条核心规则编码
3. 每个业务缺陷都应说明违反了哪条规则

## 13. 当前结论

第一期建议把本文档视为业务规则主基线，后续如有规则变更，优先追加版本记录，而不是直接在需求分析或页面文档中零散修改。
