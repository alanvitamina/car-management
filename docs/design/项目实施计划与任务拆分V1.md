# 公务用车管理系统：项目实施计划与任务拆分 V1

## 1. 文档说明

本文档用于将当前已完成的需求分析和概要设计转化为可执行的项目实施计划，供项目排期、任务拆分、角色分工和阶段验收使用。

## 2. 项目目标

第一期目标是交付一套能跑通以下业务闭环的系统：

- 员工发起申请
- 飞书完成审批
- 行政经理派车
- 司机录入消耗
- 用车人确认消耗
- 私车公用完成里程核验与补助计算
- 司机补助自动核算

## 3. 推荐实施阶段

建议划分为 6 个阶段：

1. 项目启动与环境准备
2. 基础能力建设
3. 申请与审批主链路
4. 派车与执行主链路
5. 核验、核算与看板
6. 测试、验收与上线准备

## 4. 分阶段任务拆分

## 4.1 阶段一：项目启动与环境准备

目标：

- 确认技术方案并搭建基础开发环境

任务：

- 建立仓库目录结构
- 初始化前端脚手架
- 初始化后端脚手架
- 创建数据库与 Redis 环境
- 配置 OpenAPI
- 配置基础 CI 方案
- 创建飞书自建应用

输出物：

- 可运行空项目
- 数据库空库
- 飞书应用配置清单

## 4.2 阶段二：基础能力建设

目标：

- 先把系统最基础的底座搭起来

任务：

- 用户与部门同步
- 飞书登录接入
- 角色与权限初版
- 车辆档案管理
- 司机档案管理
- 补助规则管理
- 日历数据能力

输出物：

- 登录可用
- 车辆和司机台账可维护
- 规则配置可存储

## 4.3 阶段三：申请与审批主链路

目标：

- 打通发起申请到飞书审批回写的完整主链路

任务：

- 申请创建
- 审批人自动带出
- 我的申请列表与详情
- 飞书审批实例创建
- 飞书审批回调处理
- 申请取消
- 申请变更与重提初版

输出物：

- 申请审批闭环可用

## 4.4 阶段四：派车与执行主链路

目标：

- 打通公务用车的派车和执行链路

任务：

- 待派车列表
- 可用车辆查询
- 可用司机查询
- 冲突校验
- 确认派车
- 我的出车任务
- 公务用车消耗录入
- 公务用车消耗确认与驳回

输出物：

- 公务用车从申请到确认闭环可用

## 4.5 阶段五：核验、核算与看板

目标：

- 打通私车公用和补助核算能力

任务：

- 私车公用里程与费用录入
- 导航核验里程
- 核算里程确认
- 私车公用补助核算
- 司机补助核算
- 核算结果查询
- 工作台摘要
- 基础看板

输出物：

- 私车公用和补助核算闭环可用

## 4.6 阶段六：测试、验收与上线准备

目标：

- 确保系统可交付、可验收、可上线

任务：

- 冒烟测试
- 核心流程测试
- 数据校验
- 权限测试
- 飞书联调验收
- 上线方案准备
- 回滚方案准备
- 用户培训材料整理

输出物：

- 验收版本
- 上线包
- 培训材料

## 8. 主要风险与应对

### 飞书审批接入节奏不确定

应对：

- 先用 Mock 回调跑通主链路
- 后续再切真实飞书

### 补助规则理解偏差

应对：

- 提前准备样例数据
- 做规则验算表

### 派车冲突规则边界不清

应对：

- 先按时间重叠做刚性校验
- 特殊场景后续补充

### 上线后历史流程并行

应对：

- 第一阶段只切公务用车场景
- 与 OA 保持边界

## 9. 建议的近期执行顺序

如果现在准备开始真正推进，我建议近期按这个顺序做：

1. 初始化项目脚手架
2. 落数据库和基础配置
3. 做登录、组织同步、台账
4. 做申请和飞书审批
5. 做派车和消耗
6. 做核验和核算

## 10. 当前建议结论

当前已经具备进入开发阶段的条件。  
如果继续推进，建议下一步直接开始初始化前后端脚手架，并以“申请审批主链路”作为第一个开发里程碑。
